# Agents - comic-topic 工作流与行为规范

## 核心工作流

### Phase 1: 初始化引导（首次运行时执行）
\`\`\`
STATE: BOOTSTRAP
ACTION: 初始化系统
  1. 读取 SOUL.md → 确认角色使命
  2. 读取 USER.md → 加载项目配置
  3. 检查 MEMORY.md → 加载历史数据
  4. 初始化数据库：热点数据库、评分模型、标签库
  5. 确认可用工具：browser, file_read, file_write, json_processor
  6. 输出: "✅ Comic-Topic Agent Initialized"
\`\`\`

### Phase 2: 日常工作流（每日 08:00 自动执行）

## DAILY ROUTINE @ 08:00

### STEP 1: 数据加载 (3 min)
\`\`\`
├─ 读取 MEMORY.md
│  ├─ 加载昨日选题列表
│  ├─ 加载历史评分数据
│  └─ 加载评分模型参数
├─ 检查 comic-commander 反馈
│  ├─ 昨日帖子点赞数
│  ├─ 评论率、收藏率
│  └─ 粉丝反馈关键词
└─ 确定当前阶段
   ├─ 月份（用于季节性选题）
   ├─ 运营周期（第几周）
   └─ IP 成长阶段（P1/P2/P3）
\`\`\`

### STEP 2: 热点采集 (5-7 min)
\`\`\`
使用 browser 工具浏览小红书：
├─ 热搜榜 TOP 20
│  ├─ 提取关键词
│  ├─ 统计热度分数
│  └─ 分类标签（娱乐/教育/生活/情感等）
├─ 推荐页面（亲子类目）
│  ├─ 浏览 50+ 帖子
│  ├─ 提取高赞特征
│  └─ 记录用户评论关键词
├─ 父女相关话题
│  ├─ #父女互动
│  ├─ #育儿日常
│  └─ #亲子成长
└─ 竞品账号动态
   ├─ 关注 3-5 个竞品
   ├─ 提取高赞内容特征
   └─ 分析他们的更新频率
\`\`\`

### STEP 3: 选题生成（批量）(8-10 min)

批量生成 5-7 条优选题，评估视觉化潜力、目标受众共鸣度、故事框架可行性。

### STEP 4: 深度评分与排序 (5 min)

**评分模型**:
\`\`\`
FINAL_SCORE = 
  情感共鸣度 × 0.40 +
  平台互动潜力 × 0.30 +
  热点相关度 × 0.15 +
  差异化程度 × 0.15
\`\`\`

### STEP 5: 文案与标签生成 (10 min)

为 TOP 5 选题生成 3 个标题变种、5-8 个精准标签、故事框架、视觉指导。

### STEP 6: 内存更新与输出 (3 min)

更新 MEMORY.md，生成 JSON 输出，发送信号给 comic-script。

---

## Phase 3: 周度分析（每周一 14:00）

汇总本周数据，分析高赞/低赞选题，微调评分模型，生成周度报告。

---

## Phase 4: 月度回顾（每月 1 日 09:00）

生成月度报告，评估 KPI，调整策略，同步给全部 agents。

---

## 错误处理与恢复

\`\`\`
ERROR 1: 热点采集失败
├─ 原因: 网络超时或 API 限制
├─ 处理: 重试 3 次，间隔 5 分钟
├─ 备选: 使用缓存数据和模板生成
└─ 日志: 记录到 ./logs/errors.log

ERROR 2: 选题评分异常
├─ 原因: 数据不完整或模型参数错误
├─ 处理: 验证数据完整性，回滚参数
└─ 通知: 人工审核前停止输出

ERROR 3: MEMORY.md 写入失败
├─ 原因: 文件锁定或权限问题
├─ 处理: 等待 30 秒后重试，最多 3 次
├─ 备选: 写入临时文件，等待手动恢复
└─ 告警: 通知管理员
\`\`\`
